Section 3

Node Editor

Select a node to edit its branches — each branch carries the probability on the line leading into its child node — plus its consequence/probability model and metadata. A node's own incoming probability is edited on its parent. Only the parameters required by the chosen distribution are shown, each with guidance on when to use it.

Section 4

Counter Measures

A library of countermeasure records organized into tabs. Each worksheet named Data_<TabName> becomes a tab here. Every record carries a probability of detection (PD) and fields through PD Notes. In the Node Editor, set a branch's Probability Distribution to From Countermeasure tab and pick the tab. When you run the model, it sweeps that tab's rows — one full simulation per row — and reports a per-row results table. Fields after PD Notes are dropped on import.

Import countermeasures

Each worksheet named Data_<TabName> becomes a separate tab (the name after Data_). Each row is one countermeasure; Scenario Abbreviation identifies the row and PD supplies its probability of detection. Only columns through PD Notes are kept; everything after is dropped. A CSV imports as a single tab.

Section 4

Decision Tree View

A topological diagram of the active scenario. Arrows run from parent to child; dashed arrows mark alternate "if parent does not occur" branches. Red flags broken links or cycles. Overlay simulation results to size each node by its share of impact, then export the diagram as SVG, PNG, Mermaid, or Graphviz.

After simulation each node shows P(occur) — the probability the node occurred — and E[value] — its expected (mean) contribution. Fill intensity tracks the node's share of the mean total, and the bar under each node shows its occurrence rate.

Export chain diagram

SVG and PNG capture the diagram exactly as shown, including any results overlay. Mermaid and Graphviz export the structure as text you can paste into docs, wikis, or other diagram tools.
Section 5

Simulation Settings

Iteration count, random seed, and the thresholds used for exceedance probabilities in the results.

Engine

A fixed seed makes a run fully reproducible. Change the seed to see how much results move purely from sampling noise.

Thresholds

Probability of the selected scenario metric exceeding each value is reported in Results. Comma-separate up to six.

The selected mode controls result labels, QA checks, report language, and template defaults. Probability and detection modes treat terminal probability nodes as end-state likelihoods rather than additional downstream gates.
